\title{JxMVC: a zero-dependency, security-hardened MVC framework for
  Jakarta EE in 253\,KB --- design, implementation and empirical

\keywords{web framework | Jakarta EE | zero-dependency | security-by-default |

\abstract[Abstract]{Modern ``batteries-included'' web frameworks trade a
  large runtime footprint, deep transitive-dependency trees, and a broad
  attack surface for developer convenience. We present \textbf{JxMVC}, a
  full-stack MVC framework for Jakarta EE that fits in \textbf{253\,KB with
  zero runtime dependencies} and applies \textbf{security-by-default at the
  request-pipeline level} rather than as opt-in configuration. We evaluate
  JxMVC against Spring Boot, Quarkus, Micronaut, and Javalin---plus a GraalVM
  native-image build---using a reproducible, containerised, single-command
  harness under pinned-core isolation on bare-metal Linux. JxMVC produces the
  \emph{smallest JVM-mode deployable image} of the group, starts faster than
  Spring Boot and Micronaut, and sustains competitive throughput with
  indistinguishable tail latency, serving \textbf{92.8 million requests with
  zero errors}; its one clear trade-off is higher resident memory from the
  embedded servlet container. The contribution is not a new technique but a
  \emph{design point}---minimalism, hardening-by-default, and pedagogical
  readability---backed by an honest empirical evaluation and a reusable
  open-source artifact.}

Our contributions are:
\begin{enumerate}
  \item \textbf{JxMVC}, a zero-dependency (253\,KB) full-stack MVC framework
    for Jakarta EE covering routing, DI, connection pooling, caching,
    scheduling, WebSocket, OpenAPI, validation, metrics, and OIDC.
  \item A \textbf{security-by-default pipeline design} (not opt-in) and its
    library-free implementation.
  \item A \textbf{complete OIDC client} (OAuth 2.0 + PKCE + id\_token
    signature verification) built with the JDK alone.
  \item A \textbf{reproducible, honest empirical evaluation} against four
    mainstream frameworks plus a GraalVM native build, with a one-command
    Docker harness as a reusable artifact.
  \item An \textbf{open-source artifact} verified by a 347-check
    dependency-free test suite, archived with a DOI.
\end{enumerate}
