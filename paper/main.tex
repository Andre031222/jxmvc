\title{JxMVC: a 253\,kB zero-dependency, security-by-default MVC
  framework for Jakarta EE --- design decisions, implementation
  experience, and empirical evaluation}

\keywords{web framework | Jakarta EE | zero-dependency | security-by-default |

\abstract[Abstract]{Modern ``batteries-included'' JVM web frameworks trade
  large runtime footprints, deep transitive-dependency trees, and broad
  attack surfaces for developer convenience. We present \textbf{JxMVC}, a
  full-stack MVC framework for Jakarta~EE~10 that occupies \textbf{253\,kB
  with zero runtime dependencies} and enforces \textbf{security-by-default
  structurally}, in a fixed fifteen-stage request pipeline. Unlike mainstream
  frameworks where hardening is opt-in, JxMVC applies input sanitisation,
  rate limiting, CSRF protection, authentication, and body-size caps to
  \emph{every} request by construction. Using a reproducible, containerised,
  single-command benchmark harness under pinned-core isolation on bare-metal
  Linux, we compare JxMVC against Spring Boot, Quarkus, Micronaut, and
  Javalin (plus a GraalVM native-image reference). JxMVC produces the
  \emph{smallest JVM-mode deployable image} of the group, starts faster than
  Spring Boot and Micronaut, and sustains competitive throughput with
  statistically indistinguishable tail latency while serving \textbf{92.8
  million requests with zero errors}; its principal trade-off is higher
  resident memory from the embedded servlet container. Beyond the numbers, we
  report the design decisions and implementation experience of building every
  subsystem from scratch. The contribution is a deliberate \emph{design
  point}---radical minimalism, hardening-by-default, and pedagogical
  readability---supported by honest empirical evidence and a reusable
  open-source artifact.}
